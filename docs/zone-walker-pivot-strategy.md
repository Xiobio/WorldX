# Zone Walker 转型策略：从 demo 到能挣钱的 AI 游戏

> 把 Zone Walker 当前形态（一个艾欧尼亚 walking sim demo）转向 "AI 推理肉鸽 + BYO 订阅 fallback" 的具体路线图。

---

## 一、现状定位

### 这是什么

**一个 "AI 生成的 walking sim 雏形"**，技术形态独特，但游戏成熟度极低。

**实际可玩内容（按职业标准量化）**：

| 维度 | Zone Walker | 同类成熟游戏（《Journey》《Gris》《Florence》） |
| --- | --- | --- |
| 初见到通关时长 | ~20–30 分钟 | 2–6 小时 |
| 核心动词 | 1（走 + 探查） | 3–5（走 + 互动 + 解谜 + 战斗 + 收集） |
| 动画状态数 | Yasuo ~8 个 | 每个角色 30–80 个 |
| 音频资产 | 程序合成 BGM | 数百条编排音乐 + 音效 + 台词 |
| 叙事载体 | 14 段独白 + 一段引子 | 完整三幕戏剧结构 + 角色弧光 |
| 制作工时 | ~4–8 周个人 + AI | 同类商业作 8–30 人月 |
| 重玩理由 | 无 | 章节解锁、隐藏路径、社区内容 |

**核心判断**：当前版本完成度 = 成熟游戏的 **5%–8%**。它是一个"概念证明 demo"，不是一个"游戏"。

## 二、优势与劣势分析

### 优势（应保留并放大）

1. **视觉密度**：42 张精美 chunk，每张都能讲故事。视觉信息量 / 开发时间比是革命性的
2. **零门槛分发**：单页 HTML，140 KB 代码 + 200 MB 资产，浏览器即开即玩。Web 游戏的天然优势
3. **音画基础协调**：五声音阶 + 暖色调画面，气氛对了
4. **技术新颖性**：AI 生成的走路掩码 pipeline 是原创性突破，业内尚无标准方案
5. **可复用 pipeline**：chunk 生成 + walkable mask + LLM-NPC 雏形

### 劣势（必须正视）

1. **游戏循环缺失** —— 头号问题
   - 玩家走完 14 个地标后，**游戏结束没有任何反馈**
   - 没有结局、没有积分、没有解锁
   - 违反游戏最基础契约："我投入时间，你给我闭环"

2. **操作只有一个动词**
   - 本质上玩家只在做"位置变化"
   - 没有"决策"、"风险"、"选择"

3. **风格漂移问题被掩盖**
   - 42 张 chunk 拼接仍有 AI 的不一致
   - 长时间游玩会无意识感到"不对劲"，毁沉浸感

4. **法律风险**
   - 亚索 + LoL 世界观是 Riot 的 IP
   - 商业化或大规模推广 = 收律师函
   - 必须在某个时间点切换到原创角色

5. **缺乏"再来一次"的理由**
   - 走完地图 = 看完一本画册
   - 没有 NG+、难度、隐藏内容、动态事件

6. **UX 细节欠债**
   - 无教程引导
   - 设置面板功能少
   - 无暂停（除了 settings）
   - 无读档界面
   - 无截图分享功能（walking sim 的核心传播力）
   - 无无障碍选项

## 三、三条转型路径对比

### 路径 A：把 Zone Walker 做到"独立 walking sim"水平

**目标用户**：Steam / itch.io 上喜欢《Gris》《Journey》《Inside》《Sable》的人群，约 200–500 万核心用户。

**6 个月路线图**：

| 阶段 | 任务 |
| --- | --- |
| Month 1–2 | 写一个清晰的故事框架；加 5–10 个 NPC；加结局序列；把亚索换成原创角色 |
| Month 3–4 | 加解谜机制；加 3–5 个动起来的场景元素；找音乐人花 ¥3–5 万写 6–8 段原创 BGM；全局 polish |
| Month 5–6 | 加截图分享；中英双语；上 itch.io 免费 + Steam $4.99；找 5–10 个 YouTuber / B 站 UP 试玩 |

**预期**：6 个月做到 Journey 5% 的水平 —— 但这个 5% 足够拿 1–5 万核心玩家。

**致命问题**：独立开发者一个人做不到艺术家级的 polish，结果会"看起来像 Journey 但不是 Journey"。

### 路径 B：放弃做"完整游戏"，做"AI-native 体验"

**目标用户**：对 AI 好奇的科技 / 艺术圈，初期 5 万 – 50 万人，传播力极强。

**核心思路**：做一个**只有 AI 能做的东西**。把 Zone Walker 升级成：

- **玩家描述想去的地方** → AI 实时生成 chunk → 走进去
- 每次开局 zone 都不一样
- LLM 驱动的 NPC：每个 NPC 都是 GPT / Claude，能真实对话
- TTS 实时配音
- 玩家的旅程被生成为一段视频结局，可分享

**3 个月可做出原型，6 个月可上线。**

**预期**：上 Reddit、Hacker News、Twitter 热门。短期话题度 >> 路径 A。但留存差 —— 玩家玩一次就走。

**风险**：API 成本（每个用户 session $0.50–2），需要订阅或广告变现。

### 路径 C：放弃 C 端，做工具卖给 B 端

**目标用户**：独立游戏开发者、剧本杀工作室、教育公司。

把 Zone Walker pipeline 包装成 SaaS：

- 输入：世界设定描述
- 输出：可玩的浏览器游戏 + 资产包

**最稳定盈利**但**用户增长慢**的路径。

**致命问题**：需要销售能力和企业人脉，与技术背景不匹配。

## 四、★ 推荐路径：AI 推理肉鸽（路径 B 的精化版）

综合开发可行性、商业模型、Zone Walker 资产复用度，这是最优解。

### 形态

```text
单机短局肉鸽
单局：30–50 min（一个案件）
累计：100–500 小时（数百案件）
AI 表现层：场景 + 嫌疑人肖像 + 证物 + 对话 + 配乐
玩家玩法层：调查、询问、推理、连线
```

### Zone Walker 资产 70% 可直接复用

| Zone Walker 现有 | 推理肉鸽中变成 |
| --- | --- |
| 42 张 chunks | 42 个犯罪现场房间 |
| walkable mask | 可调查物品热区 |
| 14 个 landmark + 独白 | 14 个嫌疑人 + 证词 |
| LLM-NPC 雏形 | 嫌疑人审讯系统 |
| 五声音阶 BGM | 悬疑配乐 |
| Yasuo（要换） | 原创侦探角色 |

### 商业模型

```text
🎮 基础版 ¥39（一次性）
   ├─ 完整游戏 + 30 个手作案件
   ├─ 7 天免费 BYO 试用
   └─ 无限案件需选下面两个

🔑 BYO 模式 ¥0（免费解锁）
   ├─ 自带 OpenAI/Anthropic key
   └─ 无限生成

📅 订阅 ¥29/月
   ├─ 我们的 key，每月 30 个新案件
   ├─ 云存档同步
   └─ 优先生成

💎 风格包 ¥18/包
   ├─ 黑色侦探
   ├─ 武侠悬案
   ├─ 太空法医
   └─ 维多利亚谜案
```

### 预期数据

| 指标 | 估算 |
| --- | --- |
| 加权 ARPU | ¥15–25 / 月 / 用户 |
| AI 边际成本 | ¥7 / 月 / 用户 |
| 净利润 | ¥8 / 月 / 用户 |
| LTV | ¥300–800 / 用户 |
| TAM | 500 万 – 2000 万人 |

按 1% 转化率：

```text
500 万 TAM × 1% × ¥180/年 = ¥900 万 / 年（保守）
2000 万 TAM × 1% × ¥180/年 = ¥3.6 亿 / 年（激进）
```

### Killer Hook

**「每日一案」** —— 每天 AI 自动生成一个新案件，全球玩家比赛谁最快破案，排行榜。

这是**最容易做出 demo、最容易传播、最容易留存**的方向。

## 五、分阶段路线图

### 第 0–3 月：做 MVP，免费上线验证

**目标**：跑通完整循环，找到 100–500 个早期用户。

**任务清单**：

- [ ] 写 5 个手作案件（不依赖 AI 生成）
- [ ] 改造 Zone Walker UI：调查模式 + 询问对话 + 推理板
- [ ] 接入 LLM-NPC 嫌疑人审讯
- [ ] BYO 配置面板 + 免费 7 天试用
- [ ] Stripe 订阅集成
- [ ] 把亚索换成原创侦探角色
- [ ] 上线 Vercel / 自有域名

**关键决策**：免费上线（基础版定价 ¥0）拿用户和反馈，不指望 0–3 月赚钱。

### 第 3–6 月：找早期付费用户

**目标**：第一批 100 个付费订阅 / 风格包购买。

**任务清单**：

- [ ] 上 Reddit r/gamedev、r/proceduralgeneration、r/detective、Hacker News
- [ ] B 站 / TikTok 短视频
- [ ] 收集用户反馈，迭代核心循环
- [ ] 出第一个付费风格包
- [ ] 加排行榜 + 每日案件
- [ ] Discord 社区

**关键指标**：DAU、单 session 时长、案件完成率、订阅转化率。

### 第 6–12 月：扩展规模

**目标**：年化 ¥100–500 万收入。

**任务清单**：

- [ ] 出 3–5 个风格包
- [ ] 接 B 端通道：剧本杀店、企业团建
- [ ] 加创作者经济：玩家自制案件上架抽成
- [ ] 多语言支持（英、日、韩）
- [ ] 移动端适配

### 第 12–24 月：决定主线

根据数据决定：

- 数据好 → All-in C 端，融资 / 团队扩张
- 数据中 → 双轮驱动：B 端稳定 + C 端打磨
- 数据差 → 转向 B 端 SaaS（剧本杀店 / 教育）

## 六、需要回答的开放问题

### 美学风格

- 哥特维多利亚（黑暗、阴沉、雨夜伦敦）
- 日式昭和悬疑（电车、霓虹、雪夜旅馆）
- 武侠悬案（江湖、客栈、雪山）
- 现代 / 黑色电影（雨夜街角、烟雾酒吧）

**建议**：第一个版本聚焦一种风格，把它做到极致。其他风格作为风格包。

### 案件复杂度

- 5 个嫌疑人 vs 10+ 个？
- 1 起案件 vs 多起串联？
- 必须破案才能进下一关 vs 可以"猜错就猜错"？

**建议**：MVP 阶段单案件、5 嫌疑人、必破。后期再扩展。

### 推理交互形式

- 文字选择题（最简单）
- 时间线拖拽
- 关系图连线（《Obra Dinn》风格）
- 自然语言陈述（最难，但最 AI-native）

**建议**：MVP 用文字选择题验证玩法，迭代过程中尝试关系图连线。

### 是否保留地图行走

- 完全保留（继承 Zone Walker 体验）
- 简化为热点点击
- 完全文字驱动

**建议**：保留简化版地图行走作为"现场踩点"环节，调查 / 询问环节切换到 UI 视角。

## 七、决策的根本原则

1. **AI 做表现层，人做玩法和数值** —— AI 生成案件文本、肖像、配乐；人写推理逻辑校验、UI、平衡
2. **从第一天就按 SaaS 设计** —— 订阅系统、成本上限、数据看板
3. **单机内核 + 异步社交层** —— 不要做实时多人
4. **15 分钟极致体验** —— 短局密度高于长局沉浸
5. **BYO + 订阅 fallback** —— 不是单选，而是必须共存
6. **永远警惕病毒爆红** —— 不是好事，是炸弹
