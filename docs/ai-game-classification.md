# AI 游戏 vs 传统游戏形态：分类框架

> 总结一个判断"AI 游戏应该做什么形态"的清晰坐标系。把每个传统游戏类型都映射到对应的 AI 版本，并评估 BYO + 订阅 fallback 商业模型的适配度。

---

## 一、核心判断标准

任何品类要不要做成 AI 游戏，先问两个问题：

1. **乐趣是否独立于画面精度？** ——
   答 yes 才值得考虑。《杀戮尖塔》画面 200 行像素照样卖 1500 万份；《巫师 3》画面降一档立刻失去一半价值。
2. **AI 在这个品类里是 10x 增强，还是只是装饰？** ——
   答 10x 才有意义。判定标准：把 AI 拿掉，这个游戏还能不能存在？若答**不能**或**会变成完全另一个游戏**，AI 就是 10x 增强；否则只是装饰。

---

## 二、坐标系：两个维度划出四象限

```
                          单局时长（决定 BYO 配置 ROI）
                ←———— 短局（5–30 min）————长局（1 h+）—————→
            │
        联  │   象限 ②                  象限 ①
        网  │   异步对战 / 每日挑战       MMO / 养成 / 沙盒
            │   Wordle, Among Us         FFXIV, 原神, 旅蛙
            │
            │   象限 ③                  象限 ④
        单  │   Roguelite / 卡牌构筑     模拟经营 / CRPG / 沙盒
        机  │   Slay the Spire, Balatro  RimWorld, BG3, Minecraft
            │
            ↓
```

两个轴的选择理由：

- **单局时长**：决定 BYO 模式 ROI。短局玩家不会为 5 分钟体验配置 API key；长局玩家配置一次就值回票价。
- **单机 / 联网**：决定 AI 调用的延迟敏感度。实时联网对生成速度要求极高（5 秒生图是劝退），异步联网/单机则可宽容。

## 三、四象限完整对照表

### 象限 ① — 联网长局（持续养成 / MMO 类）

| 维度 | 数据 |
| --- | --- |
| 单次会话 | 30 分钟 – 数小时 |
| 累计时长 | 数百至数千小时 |
| 玩家心态 | "我的"账号、"我的"角色、"我的"城邦 |
| 收入模型 | 月费订阅 / 抽卡 / 季票 |

**传统代表**：《魔兽世界》《最终幻想 14》《原神》《明日方舟》《动物森友会》《旅行青蛙》《Stardew Valley》（轻联网）

**AI 版本对照**：

| 传统游戏 | AI 版本 | 核心机制 |
| --- | --- | --- |
| 抽卡 RPG | **AI 卡牌养成**：每个角色都是 LLM-NPC，有持续记忆，对话永远新鲜 | 抽到 → 培养 → 对话 → 记忆累积 |
| 旅行青蛙 | **AI 伙伴**：小蛙真的会写信，明信片由 AI 生成，地点真实存在 | 极低互动 + 高情感价值 |
| 模拟人生 | **AI 城镇**：管理一个 50 人小镇，每个 NPC 都有 LLM 驱动的真实人格 | 涌现叙事 + 居民间随机戏剧 |
| 动物森友会 | **AI 邻居**：和"活着"的 NPC 长期相处，他们记得你说过的每一句话 | 慢节奏 + 持续陪伴 |

## 五、TOP 3 推荐形态深度对比

把象限 ① 和 ③ 的最佳代表，加上一个 ④ 的变种，做最终决策对比。

### 推荐 A：**AI 侦探 / 推理肉鸽**（对照 Slay the Spire）

```text
游戏形态：单机短局肉鸽
单局：30–50 min（一个案件）
累计：100–500 小时（数百案件）

商业模型：
  - 买断 ¥39（含 30 个手作案件 + 引擎）
  - BYO 模式：无限生成（重度玩家）
  - 订阅 ¥29/月：每月 30 个新案件（休闲玩家）
  - 风格包：¥18/包

预期 ARPU：¥15–25/月（含买断摊销）
预期 LTV：¥300–800/用户
```

**优势**：开发周期最短（4–8 个月可上线）；BYO ROI 最高（玩家会真的跑 200 个案件）；视觉资产可直接复用既有 chunks pipeline。

**劣势**：推理品类用户基数中等；AI 案件逻辑严密性是开发难点。

---

### 推荐 B：**AI 伴侣 / AI 角色养成**（对照 Character.AI + 旅蛙）

```text
游戏形态：联网长局养成
单局：5–30 min × 每日多次
累计：千小时级别（多年陪伴）

商业模型：
  - 客户端免费
  - BYO 模式：无限对话
  - 订阅 ¥39/月：每天 200 次互动 + 云端备份
  - 高级角色 ¥9–39/解锁
  - 创作者经济：用户卖角色，平台抽 30%

预期 ARPU：¥25–50/月
预期 LTV：¥1000–5000/用户（粘性极高）
```

**优势**：用户粘性极高（情感投入）；创作者经济解锁第二增长曲线；TAM 巨大（Character.AI 估值 $50 亿证明赛道）。

**劣势**：Character.AI 已占据先发；内容审核 / 成人边界问题；需要持久记忆架构（向量库 + LLM 长上下文）。

---

### 推荐 C：**AI 王朝 / 文明模拟**（对照 RimWorld + CK3）

```text
游戏形态：单机长局模拟经营
单局：1–5 h × 中等频率
累计：200–1000 小时

商业模型：
  - 买断 ¥69（含基础内核）
  - BYO 模式：AI 事件叙述、NPC 对话
  - 订阅 ¥39/月：托管模式 + 跨设备
  - 大型 DLC：¥38–68（新时代、新地图）

预期 ARPU：¥10–20/月
预期 LTV：¥500–2000/用户
```

**优势**：高端硬核用户付费意愿极强；Mod 社区天然形成；上限最高（可类比 CK3、矮人要塞）。

**劣势**：开发周期 18–36 个月；数字模拟内核是非 AI 工作；独立开发者几乎做不出来。

## 七、决策建议

### 如果是独立开发者 + 有可复用资产 + 想 12 个月内挣到第一笔钱

→ **选 A：AI 推理肉鸽**

对照 Slay the Spire 这种"独立爆款"模板，一个人/小团队可做但天花板高。BYO + 订阅 fallback 完美适配 —— 重度推理玩家用 BYO 省成本，轻度玩家用订阅图省心。这个细分品类目前没有强力竞品。

### 如果愿意接受 24–36 月慢长跑、能承受 12 个月零收入

→ **选 C：AI 王朝模拟**

这是真正能做出"AI 游戏代表作"的形态，但风险极高。

### 关于 B（AI 伴侣）

赛道太挤，下一波拿到融资的会是有大模型自研能力的团队，独立开发者很难突围。

## 附录：单机 vs 联网的二阶选择

四象限选定之后，还要做一个二阶决策：**社交层是实时同步还是异步沉淀？**

| | 核心：单人体验 | 核心：多人交互 |
| --- | --- | --- |
| **实时同步** | 单机 + 看朋友直播（≈纯单机） | 实时多人（Among Us、剧本杀） |
| **异步沉淀** | **★ 单机 + 异步社交层 ★** | 异步多人（Wordle、宝可梦 GO） |

**★ 那一格就是 AI 游戏的甜蜜点。**

实践上，Zone Walker 走推理肉鸽路线时，应当采用 **"单机内核（玩的时候离线） + 极简联网层（每日内容 + 排行榜 + 分享）"** 的混合结构 —— 与 Wordle、Balatro Daily、Slay the Spire Daily Climb 同模式。

需要的联网功能极少：
1. 每日内容下发（一个 GET 接口）
2. 分数上报 + 排行榜（一个 POST + 一个 GET）
3. 分享链接生成（一个 POST）

100 行后端 + 一个 Supabase 即可，比"实时多人"少 1000 倍工作量，获取 80% 社交价值。
